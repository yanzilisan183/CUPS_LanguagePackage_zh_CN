概述

  CUPS 光栅 API 提供了一个标准接口，用于读取和写入用于打印到光栅打印机的 CUPS 光栅流。由于光栅格式会不时更新，因此使用此 API 以避免与较新版本的 CUPS 不兼容非常重要。

  两种 CUPS 筛选器使用 CUPS 光栅 API － 光栅图像处理器（ RIP ）筛选器，如 pstoraster 和 cgpdftoraster （ macOS ），用于生成 CUPS 光栅文件，以及将 CUPS 光栅文件转换为打印机可用格式的打印机驱动程序筛选器。打印机驱动程序筛选器是目前最常见的。

  CUPS 光栅文件（ application/vnd.cups-raster ）由一个 RIP 筛选器生成的光栅页面描述流组成，如 pstoraster 、 imagetoraster 或 cgpdftoraster 。 CUPS 光栅文件使用 cups_raster_t 类型引用，并使用 cupsRasterOpen 函数打开。例如，要从标准输入读取光栅数据，请打开文件描述符 0 ： 

#include <cups/raster.h>

cups_raster_t *ras = cupsRasterOpen(0, CUPS_RASTER_READ);

  每页数据都以一个名为 cups_page_header2_t 的页面字典结构开始。此结构包含用于页面的颜色空间、每种颜色的位数、介质尺寸、介质类型、硬件分辨率等。

提示：

  不要将页头中的颜色空间与 PPD ColorModel 关键字混淆。 ColorModel 是指用于设备的一般颜色类型（ Gray 、 RGB 、 CMYK 、 DeviceN ），通常用于为页头以及相关颜色配置文件选择特定的颜色空间。页头颜色空间（ cupsColorSpace ）描述了颜色数据的类型和组织，例如 KCMY （黑色优先）而不是 CMYK ， RGBA （ RGB + alpha ）而不是 RGB 。

  您可以使用 cupsRasterReadHeader2 函数读取页头： 

#include <cups/raster.h>

cups_raster_t *ras = cupsRasterOpen(0, CUPS_RASTER_READ);
cups_page_header2_t header;

while (cupsRasterReadHeader2(ras, &header))
{
  /* 设置此页 */

  /* 读取光栅数据 */

  /* 完成此页 */
}

  页面字典之后是页面数据，它是一个完整分辨率，可能是代表打印机输出颜色空间中页面的压缩位图。您可以使用 cupsRasterReadPixels 函数读取未压缩的光栅数据。 for 循环通常用于一次读取页面的一行： 

#include <cups/raster.h>

cups_raster_t *ras = cupsRasterOpen(0, CUPS_RASTER_READ);
cups_page_header2_t header;
int page = 0;
int y;
char *buffer;

while (cupsRasterReadHeader2(ras, &header))
{
  /* 设置此页 */
  page ++;
  fprintf(stderr, "PAGE: %d %d\n", page, header.NumCopies);

  /* 为 1 行分配内存 */
  buffer = malloc(header.cupsBytesPerLine);

  /* 读取光栅数据 */
  for (y = 0; y < header.cupsHeight; y ++)
  {
    if (cupsRasterReadPixels(ras, buffer, header.cupsBytesPerLine) == 0)
      break;

    /* 将光栅数据写入标准输出上的打印机 */
  }

  /* 完成此页 */
}

  读取完光栅数据后，请调用 cupsRasterClose 函数以释放用于读取光栅文件的内存： 

cups_raster_t *ras;

cupsRasterClose(ras);

 CUPS 2.2/macOS 10.12 cupsRasterInitPWGHeader

  初始化 PWG 光栅输出的页头。

int cupsRasterInitPWGHeader(cups_page_header2_t *h, pwg_media_t *media, const char *type, int xdpi, int ydpi, const char *sides, const char *sheet_back);

参数

h 页头
media PWG 媒体信息
type PWG 光栅类型字符串
xdpi 交叉馈电方向（水平）分辨率
ydpi 进给方向（垂直）分辨率
sides IPP “边” 选项值
sheet_back 背面变换或 NULL 表示无

返回值

  成功为 1 ，失败为 0

详述

  “media” 参数指定要使用的媒体。

  “type” 参数指定一个 “pwg-raster-document-type-supported” 值，该值控制光栅数据的颜色空间和位深度。

  “xres” 和 “yres” 参数指定光栅分辨率，单位为每英寸点数。

  “sheet_back” 参数指定了一个 “pwg-raster-document-sheet-back” 值，用于页面背面。传入 NULL 则为正面。

cupsRasterOpen

  使用文件描述符打开光栅流。

cups_raster_t *cupsRasterOpen(int fd, cups_mode_t mode);

参数

fd 文件描述符
mode 模式 － CUPS_RASTER_READ 、 CUPS_RASTER_WRITE 、 CUPS_RASTER_WRITE_COMPRESSED 或 CUPS_RASTER_WRITE_PWG

返回值

  新的流

详述

  此函数用于将光栅流与给定的文件描述符相关联。对于大多数打印机驱动程序筛选器， “fd” 将为 0 （标准输入）。对于大多数生成光栅数据的光栅图像处理器（ RIP ）筛选器，“fd” 将为 1 （标准输出）。   写入光栅数据时，可以使用 CUPS_RASTER_WRITE 、 CUPS_RASTER_WRITE_COMPRESS 或 CUPS_RASTER_WRITE_PWG 模式 － 压缩和 PWG 输出通常要小 25-50% ，但会增加 100-300% 的执行时间开销。

cupsRasterOpenIO

  使用回调函数打开光栅流。

cups_raster_t *cupsRasterOpenIO(cups_raster_iocb_t iocb, void *ctx, cups_mode_t mode);

参数

iocb 读/写回调
ctx 回调的上下文指针
mode 模式 － CUPS_RASTER_READ 、 CUPS_RASTER_WRITE 、 CUPS_RASTER_WRITE_COMPRESSED 或 CUPS_RASTER_WRITE_PWG

返回值

  新的流

详述

  此函数将光栅流与给定的回调函数和上下文指针相关联。

  写入光栅数据时，可以使用 CUPS_RASTER_WRITE 、 CUPS_RASTER_WRITE_COMPRESS 或 CUPS_RASTER_WRITE_PWG 模式 － 压缩和 PWG 输出通常要小 25-50% ，但会增加 100-300% 的执行时间开销。

 不建议使用 cupsRasterReadHeader

unsigned cupsRasterReadHeader(cups_raster_t *r, cups_page_header_t *h);

参数

cupsRasterReadPixels

  读取光栅像素。

unsigned cupsRasterReadPixels(cups_raster_t *r, unsigned char *p, unsigned len);

参数

r 光栅流
p 指向像素缓冲区的指针
len 要读取的字节数

返回值

  读取的字节数

详述

  为了获得最佳性能，筛选器应该读取一条或多条整行。页头中的 “cupsBytesPerLine” 值可用于分配行缓冲区，并作为要读取的字节数。
