ppdc(1)

名称

  ppdc - cups ppd 编译器（已弃用）

摘要

  ppdc [-D name[=value]] [-I include-directory] [-c message-catalog] [-d output-directory] [-l language(s)] [-m] [-t] [-v] [-z] [--cr] [--crlf] [--lf] source-file

说明

  ppdc 将 PPDC 源文件编译为一个或多个 PPD 文件。此程序已弃用，将在 CUPS 的未来版本中删除。

选项

  ppdc 支持以下选项：

-D name[=value]
设置要在源文件中使用的命名变量。它相当于在源文件中使用 #define 指令。
-I include-directory
指定备用包含目录。可以提供多个 -I 选项来添加其他目录。
-c message-catalog
指定要用于本地化的 GNU gettext（ filename.po ）或 Apple strings（ filename.strings ）格式的单个消息目录文件。
-d output-directory
指定 PPD 文件的输出目录。默认输出目录为 “ppd” 。
-l language(s)
指定本地化 PPD 文件时要使用的一种或多种语言。默认语言为 “en”（英语）。使用逗号分隔多种语言，例如 “de_de 、 en_UK 、 es_es 、 es_MX 、 es_US 、 fr_CA 、 fr_fr 、 it_it ” ，将在每个文件中创建包含德语、英国英语、西班牙语（西班牙、墨西哥和美国）、法语（法国和加拿大）和意大利语的 PPD 文件。
-m
指定输出文件名应基于 ModelName 值，而不是 filename 或 PCFilenName 。
-t
指定应测试而不是生成 PPD 文件。
-v
指定详细输出，基本上是加载或写入文件的运行状态。-z 生成压缩的 PPD 文件（ filename.PPD.gz ）。默认情况下，生成未压缩的 PPD 文件。
--cr
--crlf
--lf
指定要使用的行尾 － 回车、回车和换行，或单独换行。默认情况下，仅使用换行字符。

提示

  PPD 文件已弃用，CUPS 的未来功能版本将不再支持 PPD 文件。不支持 IPP 的打印机可以使用应用程序（如 ippeveprinter(1) ）来支持。
