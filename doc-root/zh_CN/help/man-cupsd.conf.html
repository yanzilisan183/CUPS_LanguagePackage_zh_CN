cupsd.conf(5)

名称

  cupsd.conf - cups 的服务器配置文件

说明

  cupsd.conf 文件配置 CUPS 调度程序，cupsd(8)。它通常位于 /etc/cups 目录中。文件中的每一行都可以是配置指令、空行或注释。配置指令通常由一个名称和零个或多个由空格分隔的值组成。配置指令名称和值不区分大小写。注释行以 # 字符开头。

顶级指令

  以下顶级指可以被 cupsd(8) 理解：

AccessLogLevel config
AccessLogLevel actions
AccessLogLevel all
指定 AccessLog 文件的日志记录级别。“config” 级别记录添加、删除或修改打印机和类以及访问或更新配置文件的时间。当打印任务被提交、保留、释放、修改或取消，以及任何 “config” 条件时， “actions” 级别记录日志。 “all” 级别记录所有请求。默认访问日志级别为 “actions” 。
AutoPurgeJobs Yes
AutoPurgeJobs No

指定当配额不再需要任务历史记录数据时，是否自动清除该数据。默认值为 “No” 。
BrowseDNSSDSubTypes_subtype[,...]
指定要为每个共享打印机发布的 Bonjour 子类型列表。例如， “BrowseDNSSDSubTypes _cups,_print” 将告诉网络客户端 CUPS 共享和 IPP Everywhere 都受支持。默认值为 “_cups” ，这是在使用 CUPS 的系统之间进行打印机共享所必需的。
BrowseLocalProtocols all
BrowseLocalProtocols dnssd
BrowseLocalProtocols none
指定用于本地打印机共享的协议。在支持 Bonjour 的系统上默认为 “dnssd” ，否则为 “none” 。
BrowseWebIF Yes
BrowseWebIF No

指定是否发布 CUPS Web 界面。默认值为 “No” 。
Browsing Yes
Browsing No

指定是否发布共享打印机。默认值为 “No” 。
DefaultAuthType Basic
DefaultAuthType Negotiate

指定要使用的默认身份验证类型。默认值为 “Basic” 。
DefaultEncryption Never
DefaultEncryption IfRequested
DefaultEncryption Required
指定是否将加密用于经过身份验证的请求。默认值为 “Required” 。
DefaultLanguage locale
指定用于文本和 Web 内容的默认语言。默认值为 “en” 。
DefaultPaperSize Auto
DefaultPaperSize None
DefaultPaperSize sizename
指定新打印队列的默认纸张大小。“Auto” 使用特定于区域设置的默认值，而 “None” 指定没有默认纸张大小。具体尺寸名称通常为 “Letter” 或 “A4” 。默认值为 “Auto” 。
DefaultPolicy policy-name
指定要使用的默认访问策略。默认访问策略为 “default” 。
DefaultShared Yes
DefaultShared No
指定默认情况下是否共享本地打印机。默认值为 “Yes” 。
DirtyCleanInterval seconds
指定更新配置和状态文件的延迟。值为 0 会导致更新尽快发生，通常在几毫秒内发生。默认值为 “30” 。
DNSSDHostNamehostname.example.com
指定用于 Bonjour 共享的服务器的完全限定域名。默认值通常是服务器的 “.local” 主机名。
ErrorPolicy abort-job
指定应中止（放弃）失败的打印任务，除非为打印机另行指定。
ErrorPolicy retry-current-job
指定应立即重试失败的打印任务，除非为打印机另行指定。
ErrorPolicy retry-job
指定失败的打印任务应在以后重试，除非为打印机另行指定。“retry-job” 错误策略是默认策略。
ErrorPolicy stop-printer
指定失败的打印任务应停止打印机，除非为打印机另行指定。
FilterLimit limit
指定并发运行的筛选器的最大成本，可用于最小化磁盘、内存和 CPU 资源问题。限制为 0 将禁用筛选器限制。非 PostScript 打印机的平均打印需要大约 200 的筛选限制。 PostScript 打印机需要大约一半（ 100 ）。将限制设置为低于这些阈值将有效地限制调度程序在任何时候打印单个任务。默认限制为 “0” 。
FilterNice nice-value
指定为打印任务而运行的筛选器的计划优先级（ nice(8) 值）。nice 值的范围从 0（最高优先级）到 19（最低优先级）。默认值为 0 。
GSSServiceName name
指定使用 Kerberos 身份验证时的服务名称。默认的服务名称是 “http”。
HostNameLookups On
HostNameLookups Off
HostNameLookups Double
指定是否对连接的客户端执行反向查找。“Double” 设置会导致 cupsd(8) 验证从该地址解析的主机名是否与为该主机名返回的其中一个地址匹配。双重查找还可以防止具有未注册地址的客户端连接到您的服务器。默认设置为 “Off” ，以避免主机名查找时可能出现的服务器性能问题。仅在绝对需要时才将此选项设置为 “On” 或 “Double” 。
IdleExitTimeout seconds
指定由于不活动而关闭前等待的时间长度。默认值为 “60” 秒。注：仅适用于按需运行的 cupsd(8) （例如，使用 -l ）。
JobKillDelay seconds
指定在终止与已取消或保留的任务关联的筛选器和后端之前等待的秒数。默认值为 “30” 。
JobRetryInterval seconds
指定任务重试之间的间隔（以秒为单位）。这通常用于传真队列，但也可用于错误策略为 “retry-job” 或 “retry-current-job” 的正常打印队列。默认值为 “30” 。
JobRetryLimit count
指定对任务执行的重试次数。这通常用于传真队列，但也可用于错误策略为 “retry-job” 或 “retry-current-job” 的正常打印队列。默认值为 “5” 。
KeepAlive Yes
KeepAlive No
指定是否支持 HTTP 保留活动连接。默认值为 “Yes” 。
KeepAliveTimeout seconds
指定空闲客户端连接保留打开的时间。默认值为 “30” 。
<Limit operation ...> ... </Limit>
指定被限制在策略节内的 IPP 操作。IPP 操作名称列在下面的 “IPP 操作名称” 一节中。
<Limit method ...> ... </Limit>
<LimitExcept method ...> ... </LimitExcept>
指定限制在位置节内的 HTTP 方法。HTTP 方法名称列在下面的 “HTTP 方法名称” 部分中。
LimitRequestBody size
指定打印文件、IPP 请求和 HTML 表单数据的最大大小。默认值为 “0” ，禁用限制检查。
Listen ipv4-address:port
Listen [ipv6-address]:port
Listen *:port
Listen /path/to/domain/socket
侦听连接的指定地址和端口或域套接字路径。可以提供多个侦听指令来侦听多个地址。Listen 指令与 Port 指令类似，但允许您限制对特定接口或网络的访问。
ListenBackLog number
指定将允许的挂起连接数。这通常只影响已达到 MaxClient 限制的非常繁忙的服务器，但也可能由大量同时连接触发。当达到限制时，操作系统将拒绝额外的连接，直到调度程序可以接受挂起的连接为止。默认值是操作系统定义的默认限制，对于较旧的操作系统，通常为 “5” ，对于较新的操作系统，通常为 “128” 。
<Location /path> ... </Location>
指定指定位置的访问控制。路径记录在 “位置路径” 一节中。
LogDebugHistory number
指定在打印任务中发生错误时保留用于记录的调试消息数。无论日志级别设置如何，都会记录调试消息。
LogLevel none
LogLevel emerg
LogLevel alert
LogLevel crit
LogLevel error
LogLevel warn
LogLevel notice
LogLevel info
LogLevel debug
LogLevel debug2
指定 ErrorLog 文件的日志记录级别。值 “none” 停止所有日志记录，而 “debug2” 记录所有内容。默认值为 “warn” 。
LogTimeFormat standard
LogTimeFormat usecs
指定日志文件中日期和时间的格式。值 “standard” 是默认值，记录整秒，而 “usecs” 记录微秒。
MaxClients number
指定调度程序允许的最大同时客户端数。默认值为 “100” 。
MaxClientsPerHost number
指定允许来自单个地址的同时客户端的最大数量。默认值是 MaxClients 值。
MaxCopies number
指定用户可以打印每个任务的最大份数。默认值为 “9999” 。
MaxHoldTime seconds
指定在取消任务之前，任务可以保留在 “indefinite” 保留状态的最长时间。默认值为 “0” ，这将禁用取消保留的任务。
MaxJobs number
指定允许的最大同时任务数。设置为 “0” 以允许无限数量的任务。默认值为 “500” 。
MaxJobsPerPrinter number
指定每台打印机允许的最大同时任务数。默认值为 “0” ，允许每个打印机最多有 MaxJobs 任务。
MaxJobsPerUser number
指定每个用户允许的最大同时任务数。默认值为 “0” ，允许每个用户最多有 MaxJobs 任务。
MaxJobTime seconds
指定取消任务之前打印任务所需的最长时间。设置为 “0” 可禁用 “stuck” 任务的取消。默认值为 “10800”（3小时）。
MaxLogSize size
指定日志文件在轮换之前的最大大小。值 “0” 禁用日志轮换。默认值为 “1048576”（1MB）。
MultipleOperationTimeout seconds
指定多文件打印任务中文件之间允许的最长时间。默认值为 “900”（15分钟）。
<Policy name> ... </Policy>
指定命名策略的访问控制。
Port number
侦听连接的指定端口号。
PreserveJobFiles Yes
PreserveJobFiles No
PreserveJobFiles seconds
指定打印任务后是否保留任务文件（文档）。如果指定了数值，任务文件将在打印后保留指定的秒数。默认值为 “86400”（保留 1 天）。
PreserveJobHistory Yes
PreserveJobHistory No
PreserveJobHistory seconds
指定打印任务后是否保留任务历史记录。如果指定了数值，任务历史记录将在打印后保留指定的秒数。如果 “Yes” ，任务历史记录将被保留，直到达到 MaxJobs 限制。默认值为 “Yes” 。
ReloadTimeout seconds
指定重新启动调度程序之前等待任务完成的时间量。默认值为 “30” 。
ServerAdmin email-address
指定服务器管理员的电子邮件地址。默认值为 “root@ServerName" 。
ServerAlias hostname [ ... hostname ]
ServerAlias *
当客户端从外部接口连接到调度程序时，ServerAlias 指令用于 HTTP 主机头验证。使用特殊名称 “*” 会使您的系统暴露于已知的基于浏览器的 DNS 重新绑定攻击，即使是通过防火墙访问站点。如果备用名称的自动发现不起作用，建议使用 ServerAlias 指令列出每个备用名称，而不是使用 “*” 。
ServerName hostname
指定服务器的完全限定主机名。默认值是 hostname(1) 命令报告的值。
ServerTokens None
ServerTokens ProductOnly
ServerTokens Major
ServerTokens Minor
ServerTokens Minimal
ServerTokens OS
ServerTokens Full
指定 HTTP 响应的服务器标头中包含的信息。“None” 将禁用服务器头。 “ProductOnly” 报告 “CUPS” 。 “Major” 报告 “CUPS/major IPP/2” 。 “Minor” 报告 “CUPS/major.Minor IPP/2.1” 。 “Minimal” 报告 “CUPS/major.minor.patch IPP/2.1” 。 “OS” 报告 “CUPS/major.minor.path (osname osversion) IPP/2.1” 。 “Full” 报告 “CUPS/major.minor.path (osname osversion; architecture) IPP/2.1”。默认值为 “Minimal” 。
SSLListen ipv4-address:port
SSLListen [ipv6-address]:port
SSLListen *:port
侦听指定地址和端口上的加密连接。
SSLOptions [AllowDH] [AllowRC4] [AllowSSL3] [DenyCBC] [DenyTLS1.0] [MaxTLS1.0] [MaxTLS1.1] [MaxTLS1.2] [MaxTLS1.3] [MinTLS1.0] [MinTLS1.1] [MinTLS1.2] [MinTLS1.3]
SSLOptions None
设置加密选项（仅在 /etc/cups/client.conf 中）。默认情况下，CUPS 仅支持使用已知安全密码套件使用 TLS v1.0 或更高版本进行加密。当使用 Allow 选项时，安全性会降低。当使用 Deny 选项时，安全性得到增强。AllowDH 选项启用使用普通 Diffie-Hellman 密钥协商的密码套件（在使用 GNU TLS 的系统上不受支持）。 AllowRC4 选项启用 128 位 RC4 密码套件，这是一些旧客户端所必需的。 AllowSSL3 选项启用 SSL v3.0 ，这是一些不支持 TLS v1.0 的旧客户端所必需的。 DenyCBC 选项禁用所有 CBC 密码套件。 DenyTLS1.0 选项禁用 TLS v1.0 支持 － 这将最低协议版本设置为 TLS v1.1 。 MinTLS 选项设置要支持的最低 TLS 版本。 MaxTLS 选项设置要支持的最大 TLS 版本。目前并非所有操作系统都支持 TLS 1.3。
SSLPort port
侦听指定端口上的加密连接。
StrictConformance Yes
StrictConformance No
指定调度程序是否要求客户端严格遵守 IPP 规范。默认值为 “No”。
Timeout seconds
指定 HTTP 请求超时。默认值为 “900”（ 15 分钟）。
WebInterface yes
WebInterface no
指定是否启用 Web 界面。默认值为 “No” 。

Http 方法名称

  cupsd(8) 支持以下 HTTP 方法：

GET
客户端用于下载图标和其他打印机资源以及访问 CUPS Web 界面。
HEAD
客户端用于获取资源的类型、大小和修改日期。
OPTIONS
客户端用于建立安全（ SSL/TLS ）连接。
POST
客户端用于从 CUPS Web 界面提交 IPP 请求和 HTML 表单。
PUT
客户端用于上载配置文件。

CUPS-Accept-Jobs
允许打印机接受新任务。
CUPS-Add-Modify-Class
添加或修改打印机类。
CUPS-Add-Modify-Printer
添加或修改打印机。
CUPS-Authenticate-Job
释放为进行身份验证而保留的任务。
CUPS-Delete-Class
删除打印机类。
CUPS-Delete-Printer
删除打印机。
CUPS-Get-Classes
获取打印机类的列表。
CUPS-Get-Default
获取服务器默认打印机或打印机类。
CUPS-Get-Devices
获取当前可用设备的列表。
CUPS-Get-Document
获取任务的文档文件。
CUPS-Get-PPD
获取一个 PPD 文件。
CUPS-Get-PPDs
获取已安装 PPD 文件的列表。
CUPS-Get-Printers
获取打印机列表。
CUPS-Move-Job
移动任务。
CUPS-Reject-Jobs
阻止打印机接受新任务。
CUPS-Set-Default
设置服务器默认打印机或打印机类。
Cancel-Job
取消任务。
Cancel-Jobs
取消一个或多个任务。
Cancel-My-Jobs
取消用户创建的一个或多个任务。
Cancel-Subscription
取消订阅。
Close-Job
关闭正在等待更多文档的任务。
Create-Job
创建没有文档的新任务。
Create-Job-Subscriptions
创建任务事件的订阅。
Create-Printer-Subscriptions
创建打印机事件的订阅。
Get-Job-Attributes
获取有关任务的信息。
Get-Jobs
获取任务列表。
Get-Notifications
获取订阅的事件通知列表。
Get-Printer-Attributes
获取有关打印机或打印机类的信息。
Get-Subscription-Attributes
获取有关订阅的信息。
Get-Subscriptions
获取订阅的列表。
Hold-Job
保留打印中的任务。
Hold-New-Jobs
保留全部新的打印任务。
Pause-Printer
停止打印机或打印机类对任务的处理。
Pause-Printer-After-Current-Job
在当前任务完成后，停止打印机或打印机类对任务的处理。
Print-Job
使用单个文档创建新任务。
Purge-Jobs
取消一个或多个任务并删除任务历史记录。
Release-Held-New-Jobs
允许打印以前保留的任务。
Release-Job
允许打印任务。
Renew-Subscription
续订。
Restart-Job
如果可能，重新打印任务。
Send-Document
将文档添加到任务中。
Set-Job-Attributes
更改任务信息。
Set-Printer-Attributes
更改打印机或打印机类信息。
Validate-Job
验证新任务的选项。

位置路径

  配置cupsd(8) 时通常使用以下路径：

/
所有获取操作（ 获取打印机、获取任务等）的路径
/admin
所有管理操作（添加打印机、删除打印机、启动打印机等）的路径
/admin/conf
访问 CUPS 配置文件（ cupsd.conf 、 client.conf 等）的路径
/admin/log
访问 CUPS 日志文件的路径（访问日志、错误日志、页面日志）
/classes
所有打印机类的路径
/classes/name
指定打印机类的资源
/jobs
所有任务（保留任务、释放任务等）的路径
/jobs/id
指定任务的路径
/printers
所有打印机的路径
/printers/name
指定打印机的路径
/printers/name.png
指定打印机的图标文件路径
/printers/name.ppd
指定打印机的 PPD 文件路径

在位置和限制部分内有效的指令

  以下指令可以放在 cupsd.conf 文件的位置和限制部分中：

Allow all
Allow none
Allow host.domain.com
Allow *.domain.com
Allow ipv4-address
Allow ipv4-address/netmask
Allow ipv4-address/mm
Allow [ipv6-address]
Allow [ipv6-address]/mm
Allow @IF(name)
Allow @LOCAL
允许从命名的主机、域、地址或接口进行访问。 @IF(name) 表单使用为命名接口配置的当前子网。 @LOCAL 表单使用为所有非点对点接口配置的当前子网，例如，使用以太网和 Wi-Fi 接口，但不使用 DSL 和 VPN 接口。 Order 指令控制是在拒绝行之前还是之后计算允许行。
AuthType None
AuthType Basic
AuthType Default
AuthType Negotiate
指定所需的身份验证类型。值 “Default” 对应于 DefaultAuthType 值。
Deny all
Deny none
Deny host.domain.com
Deny *.domain.com
Deny ipv4-address
Deny ipv4-address/netmask
Deny ipv4-address/mm
Deny [ipv6-address]
Deny [ipv6-address]/mm
Deny @IF(name)
Deny @LOCAL
拒绝来自命名主机、域、地址或接口的访问。 @IF(name) 表单使用为命名接口配置的当前子网。 @LOCAL 表单使用为所有非点对点接口配置的当前子网，例如，使用以太网和 Wi-Fi 接口，但不使用 DSL 和 VPN 接口。 Order 指令控制是在允许行之前还是之后计算拒绝行。
Encryption IfRequested
Encryption Never
Encryption Required
指定特定位置所需的加密级别。默认值为 “IfRequested” 。
Order allow,deny
指定默认情况下拒绝访问。然后处理允许行，后跟拒绝行，以确定客户端是否可以访问特定资源。
Order deny,allow
指定默认情况下允许访问。然后处理拒绝行，后跟允许行，以确定客户端是否可以访问特定资源。
Require group group-name [ group-name ... ]
指定经过身份验证的用户必须是其中一个命名组的成员。
Require user {user-name|@group-name} ...
指定经过身份验证的用户必须与其中一个命名用户匹配，或者是其中一个命名组的成员。组名 “@SYSTEM” 对应于 cups-files.conf(5) 文件中的 SystemGroup 指令定义的组列表。组名 “@OWNER” 对应于资源的所有者，例如提交打印任务的人员。注意： “root” 用户不是特殊用户，必须像其他用户帐户一样被授予特权。
Require valid-user
指定任何经过身份验证的用户都是可接受的。
Satisfy all
指定必须满足所有 Allow 、 AuthType 、 Deny 、 Order 和 Require 条件才能允许访问。
Satisfy any
指定如果满足身份验证（ AuthType/Require ）或地址（ Allow/Deny/Order ）条件，则任何客户端都可以访问资源。例如，这可用于仅对远程访问要求身份验证。

在策略部分内有效的指令

  以下指令可以放在 cupsd.conf 文件的策略部分中：

JobPrivateAccess all
JobPrivateAccess default
JobPrivateAccess {user|@group|@ACL|@OWNER|@SYSTEM} ...
指定任务私有值的访问列表。“default” 访问列表为 “@OWNER @SYSTEM” 。 “@ACL” 映射到打印机的 requesting-user-name-allowed 或 requesting-user-name-denied 值。 “@OWNER” 映射到任务的所有者。 “@SYSTEM” 映射到 cups-files.conf(5) 文件中 SystemGroup 指令中列出的组。

不建议使用的指令

  以下指令已弃用，将在 CUPS 的未来版本中删除：

Classification banner

指定服务器的安全分类。可以使用任何有效的标题名称，包括 “classified” 、 “confidential” 、 “secret” 、 “topsecret” 和 “unclassified” ，或者可以省略标题以禁用安全打印功能。默认设置为无分类 banner 。
ClassifyOverride Yes
ClassifyOverride No

指定用户是否可以使用 “job-sheets” 选项覆盖单个打印任务的分类（封面）。默认值为 “No” 。
PageLogFormat format-string
指定页面日志行的格式。以百分比（%）字符开头的序列将替换为相应的信息，而所有其他字符将按字面复制。可识别以下百分比序列： 
    "%%" inserts a single percent character.
    "%{name}" inserts the value of the specified IPP attribute.
    "%C" inserts the number of copies for the current page.
    "%P" inserts the current page number.
    "%T" inserts the current date and time in common log format.
    "%j" inserts the job ID.
    "%p" inserts the printer name.
    "%u" inserts the username.

  默认值为空字符串，这将禁用页面日志记录。字符串 “%p %u %j %T %P %C %{job-billing} %{job-originating-host-name} %{job-name} %{media} %{sides}” 使用标准项创建页面日志。使用 “%{job-impressions-completed}” 插入已打印的页数（侧面），或使用 “%{job-media-sheets-completed}” 插入已打印的页数。

RIPCache size
指定将文档转换为打印机位图时要使用的最大内存量。默认值为 “128m” 。

提示

  以前允许在 cupsd.conf 文件中使用的文件、目录和用户配置指令现在存储在 cups-files.conf(5) 文件中，以防止某些类型的权限提升攻击。

  更改 cupsd.conf 文件后，必须手动重新启动调度程序。在 Linux 上，这通常使用 systemctl(8) 命令来完成，而在 macOS 上则使用 launchctl(8) 命令。

  @LOCAL 宏名称可能会令人困惑，因为运行 cupsd 的系统通常属于与其客户端不同的一组子网。

遵循

  文件格式基于 Apache HTTP 服务器配置文件格式。

例子

  以 32 MB 的最大日志文件大小记录所有内容： 

    AccessLogLevel all
    LogLevel debug2
    MaxLogSize 32m

  需要对 10. 以外的网络访问进行身份验证： 

    <Location />
    Order allow,deny
    Allow from 10./8
    AuthType Basic
    Require valid-user
    Satisfy any
    </Location>
