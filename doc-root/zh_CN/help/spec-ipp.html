IPP 的 CUPS 实现

介绍

  CUPS 实现 IPP/2.1 以及以下规范中定义的操作和属性：

  • PWG 5100.11: IPP 任务和打印机扩展 - 第2组（JPS2）（所有操作加上 “job-creation-attributes-supported” 、 “job-ids” 、 “job-password” 、 “job-password-encryption” 、 “media-col-database” 和 “which-jobs” 属性）
  • PWG 5100.13: IPP 任务和打印机扩展 - 第3组（JPS3）（除 “printer-geo-location” 和 “printer-icc-profiles” 外，IPP 所需的所有操作/属性）
  • PWG 5100.14: IPP 无处不在（符合客户和大多数打印机要求）
  • PWG 5100.15: IPP 传真输出服务（用于传真队列）
  • PWG 5100.16: 基于 IPP 事务的打印扩展
  • RFC 3995: IPP: 事件通知和订阅
  • RFC 3996: IPP: 事件通知的 “ippget” 传递方法
  • RFC 3998: IPP: 任务和打印机管理操作 (Disable-Printer, Enable-Printer, Hold-New-Jobs, 和 Release-Held-New-Jobs 操作)
  • RFC 7472: IPP over HTTPS 传输绑定和 “ipps” URI 方案

  CUPS 还提供了 17 个新操作和许多新属性，以支持单个主机上的多个 IPP 打印机和打印机类。

IPP URIs

  CUPS 支持 “http” 、 “https” 、 “ipp” 和 “ipp” 方案。使用以下资源名称：

scheme://hostname:port/
可用于所有“获取”操作和服务器订阅。
scheme://hostname:port/admin/
用于所有管理操作。
scheme://hostname:port/classes/name
指定打印机类。
scheme://hostname:port/jobs/id
指定任务。
scheme://hostname:port/printers/name
指定打印机。

  因此，典型的打印机 URI 是 “ipp://foo.example.com/printers/LaserJet” 。此外，CUPS 调度程序还支持（在启用时） “http://foo.example.com:port/” 和 “https://foo.example.com:port/” 的方式。

操作

  以下各节介绍 CUPS 支持的操作。为了简洁起见，不描述仅使用标准 IPP 属性的操作。

Cancel-Job 操作（扩展）

  Cancel-Job 操作（0x0008）取消指定的任务。CUPS 1.4 增加了对 purge-job (boolean) 操作属性的支持，该属性（如果为“true”）也会删除任务的所有历史记录和文档文件。

Purge-Jobs 操作

  Purge-Jobs 操作（0x0012）取消给定目标上的所有任务，并可选择删除任务的所有历史记录和文档文件。CUPS 1.2 增加了对 purge-job (boolean) 操作属性的支持，该属性（如果为“false”）保留已取消任务的所有历史记录和文档文件。

提示：

  应使用 “Cancel-Jobs” 和 “Cancel-My-Jobs” 操作，而不是清除任务。

CUPS 1.2/macOS 10.5Create-Printer-Subscription

  Create-Printer-Subscription 操作（0x0016）为打印机或服务器事件通知创建订阅。CUPS 除了 IPP 通知规范中的标准事件外，还提供了几个附加事件。CUPS 添加以下 notify-events (1setOf type2 keyword) 值：

  • printer-added - 在添加打印机或类时收到通知
  • printer-deleted - 在删除打印机或类时收到通知
  • printer-modified - 在修改打印机或类时收到通知
  • server-audit - 发生安全状况时收到通知
  • server-restarted - 服务器重新启动时收到通知
  • server-started - 服务器启动时收到通知
  • server-stopped - 服务器停止时收到通知

属性

  CUPS 提供许多扩展属性以支持多个设备、PPD 文件、标准任务筛选器、打印机和打印机类。

设备属性不建议使用

  CUPS-Get-Devices 操作返回设备属性，并枚举服务器支持的所有可用硬件设备和网络协议。设备属性在 printer-attributes-tag 组中报告。

device-class (type2 keyword)不建议使用

  “device-class” 属性提供设备类别，可以是以下之一：

  • 'file': 磁盘文件。
  • 'direct': 并行或固定速率串行数据端口，目前用于 Centronics、IEEE-1284 和 USB 打印机端口。
  • 'serial': 可变速率串行端口。
  • 'network': 网络连接，通常通过 AppSocket 、 HTTP 、 IPP 、 LPD 或 SMB/CIFS 协议。

device-id (text(1023))不建议使用

  "device-id" 属性提供设备的 IEEE-1284 设备 ID 字符串。

device-info (text(127))不建议使用

  "device-info" 属性指定描述设备的可读字符串，例如 “Parallel Port #1”。

device-location (text(127))不建议使用

  "device-location" 属性指定打印机的物理位置，例如“二楼计算机实验室”。

device-make-and-model (text(127))不建议使用

  "device-make-and-model" 属性指定连接到设备的打印机提供的设备标识字符串。如果设备或打印机不支持标识，则此属性包含字符串 “unknown” 。

device-uri (uri)

  "device-uri" 属性指定设备的唯一标识符。“device-uri” 字符串的实际格式取决于 “device-class” 属性的值：

  • 'file': "device-uri" 使用格式为 'file:///path/to/filename'.
  • 'direct': "device-uri" 使用格式为 'scheme:/dev/filename' 或 'scheme://vendor/identifier'，其中，在当前实现中，方案可能是 “parallel” 或 “usb” 。
  • 'serial': "device-uri" 使用格式为 'serial:/dev/filename?baud=value+parity=value+flow=value'。波特值是数据速率，单位为比特/秒；支持的值取决于底层硬件。奇偶校验值可以是 “none” 、 “even” 或 “odd” 之一。流量值可以是 “none” 、 “soft”（XON/XOFF握手）、 “hard” 或 “rts/cts”（rts/cts握手）或 “dtrdsr”（DTR/DSR握手）之一。

      CUPS-Get-Devices 返回的 URI 将包含设备支持的最大波特率和可用的最佳流控制类型（“soft” 或 “hard”）。

  • 'network': "device-uri" 使用格式为 'scheme://[username:password@]hostname[:port]/[resource]'，其中，在当前实现中，方案可以是 “http” 、 “https” 、 “ipp” 、 “lpd” 、 “smb” 或 “socket” 。

      CUPS-Get-Devices 返回的 URI 只能包含方案名称（“scheme”）。在添加新打印机时，由客户端应用程序添加适当的主机和其他信息。

      Get-Printer-Attributes 和 CUPS-Get-Printers 返回的 URI 包含任何用户名和密码信息；该信息仍然存储在服务器内部，并由服务器用于执行任何所需的身份验证。

  "page-set" 属性指定要在文件中打印的页面。支持的关键字为 “all” 、 “even” 和 “odd” 。默认值为 “all” 。

PPD 属性不建议使用

  PPD 属性在打印机属性标签组中返回。

ppd-device-id (text(127))不建议使用

  "ppd-device-id" 属性指定 PPD 文件描述的设备的 IEEE-1284 设备 ID 字符串。

ppd-make (text(127))不建议使用

  "ppd-make" 属性指定打印机的制造商（PPD 文件中的制造商属性）。如果 PPD 文件中未指定制造商，则使用 PPD 文件中的昵称属性进行有根据的猜测。

ppd-make-and-model (text(127))不建议使用

  "ppd-make-and-model" 属性指定 PPD 文件的制造商和型号名称（PPD 文件中的昵称属性）。如果 PPD 文件中未指定 make 和 model ，则使用 ModelName 或 short 昵称属性。

ppd-model-number (integer)不建议使用

  "ppd-model-number" 属性提供 PPD 文件中的 cupsModelNumber 值。

ppd-name (name(255))不建议使用

  "ppd-name" 属性指定服务器上相对于模型目录的 PPD 文件名或映射到驱动程序目录中特定驱动程序接口的 URI。使用斜杠（/）描述目录路径。

ppd-natural-language (1setOf naturalLanguage)不建议使用

  "ppd-natural-language" 属性指定 PPD 文件的语言编码（ PPD 文件中的 LanguageVersion 属性）。如果语言未知或未定义，则假定为 “en”（英语）。

ppd-product (1setOf text(127))不建议使用

  "ppd-product" 属性指定 PPD 文件中的产品属性值。

ppd-psversion (1setOf text(127))不建议使用

  "ppd-product" 属性指定 PPD 文件中的 PSVersion 属性值。

ppd-type (type1 keyword)不建议使用

  "ppd-type" 属性指定 PPD 文件描述的驱动程序类型：

  • 'fax': 传真机或多功能设备
  • 'pdf': PDF 打印机
  • 'postscript': PostScript 打印机（无筛选器）
  • 'raster': CUPS 光栅设备
  • 'unknown': 未知的或复合设备

打印机属性

auth-info-required (1setOf type2 keyword)CUPS 1.3/macOS 10.5

  "auth-info-required" 属性指定打印任务所需的身份验证信息。可以识别以下关键字：

  • 'domain': 需要域名。
  • 'negotiate': 必须使用 Kerberos － 此关键字只能单独出现，并导致 cupsd 收集打印用户的 UID。
  • 'none': 不需要身份验证 － 此关键字只能自行显示。
  • 'password': 需要密码。
  • 'username': 需要用户名。某些协议（如 SMB ）在用户名前加上域前缀，例如“域\用户”。

job-k-limit (integer)CUPS 1.1

  "job-k-limit" 属性指定用户可以打印的最大 KB 数，包括 banner 文件。默认值 0 指定没有限制。

job-page-limit (integer)CUPS 1.1

  "job-page-limit" 属性指定用户可以打印的最大页数，包括 banner 文件。默认值 0 指定没有限制。

job-quota-period (integer)CUPS 1.1

marker-change-time (integer)CUPS 1.3/macOS 10.5

marker-colors (1setof name(MAX))CUPS 1.3/macOS 10.5

marker-high-levels (1setof integer(0:100))CUPS 1.4/macOS 10.6

  "marker-high-levels" 状态属性指定指示接近满状态的供应级别。值 100 应该用于消耗/耗尽的耗材，例如墨盒。

marker-levels (1setof integer(-3:100))CUPS 1.3/macOS 10.5

  "marker-levels" 状态属性指定打印机的当前供应级别。它仅在驱动提供供电水平时可用。值 -1 表示级别不可用，-2 表示未知，-3 表示级别未知但尚未达到容量。从 0 到 100 的值表示相应的百分比。

marker-low-levels (1setof integer(0:100))CUPS 1.4/macOS 10.6

  "marker-low-levels" 状态属性指定指示接近空状态的供应级别。填充的耗材（例如废墨水罐）的值应为 0 。

marker-message (text(MAX))CUPS 1.4/macOS 10.6

marker-names (1setof name(MAX))CUPS 1.3/macOS 10.5

port-monitor" (name(127))不建议使用

  "printer-id" 状态属性提供标识打印机的唯一整数。当仅提供 IP 地址和整数来标识打印队列时，使用该选项。

printer-type (type2 enum)

  "printer-type" 状态属性指定打印机或类的打印机类型和功能位。默认值是根据打印机的内部状态信息和 PPD 文件计算得出的。定义了以下位：

位 说明
0x00000001 这是一个打印机类。
0x00000002 是一个远程的目标。
0x00000004 可以黑色打印。
0x00000008 可以彩色打印。
0x00000010 可以在硬件中进行双面打印。
0x00000020 可以装订输出。
0x00000040 可以在硬件上进行快速复制。
0x00000080 可以在硬件上进行快速拷贝整理（按份打印）。
0x00000100 可以压痕输出。
0x00000200 可以覆盖输出。
0x00000400 可以绑定输出。
0x00000800 可以对输出进行排序。
0x00001000 可以处理 US-Legal/A4 的打印介质。
0x00002000 可以处理 US-Legal/A4 到 ISO-C/A2 的打印介质。
0x00004000 可处理大于 ISO-C/A2 的介质。
0x00008000 可以处理用户定义的介质。
0x00010000 这是一个隐式（服务器生成的）类。
0x00020000 这是网络上的默认打印机。
0x00040000 这是传真机。
0x00080000 拒绝任务。
0x00100000 删除此队列。
0x00200000 队列未共享。
0x00400000 队列需要身份验证。
0x00800000 队列支持 CUPS 命令文件。
0x01000000 队列已自动发现并添加。
0x02000000 队列是没有打印功能的扫描仪。
0x04000000 队列是具有扫描功能的打印机。
0x08000000 队列是具有 3D 功能的打印机。

printer-type-mask (type2 enum)CUPS 1.1

  "printer-type-mask" 属性用于使用 CUPS-Get-Printers 和 CUPS-Get-Classes 操作选择打印机或类。位的定义与打印机类型属性相同，默认为所有 1。
