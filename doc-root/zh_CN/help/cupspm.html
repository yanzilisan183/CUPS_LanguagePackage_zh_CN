介绍

  CUPS 提供 “cups” 库与 CUPS 的不同部分和互联网打印协议( IPP )打印机进行对话。“cups” 库通过包含 <cups/cups.h> 头文件来访问库函数。

  CUPS 基于 Internet 打印协议（ “IPP” ），该协议允许客户端（应用程序）与服务器（调度程序、打印机等）通信，以获取目标列表、发送打印任务等。您可以使用指向不透明结构 http_t 的指针来确定要与哪个服务器通信。当您想与 CUPS 调度程序对话时，可以使用 CUPS_HTTP_DEFAULT 常量。

指南

  在编写软件（打印机驱动程序除外）时，使用 “cups” 库：

  • 不要使用文档未收录或不推荐的 API ，
  • 不要依赖预配置的打印机，
  • 不要假设打印机支持特定功能或格式，以及
  • 不要依赖实施细节（ PPD 等）

  CUPS 旨在将用户和开发人员与打印机和文件格式的实现细节隔离开来。其目标是允许应用程序以符合用户意图的标准格式提供打印文件（“打印四份，在 A4 介质上双面打印，并按份装订”），并让打印系统管理所需的打印机通信和格式转换。

  类似地，打印机和任务管理应用程序可以使用标准查询操作以通用形式获取状态信息，并使用标准管理操作控制这些打印机和任务的状态。

提示：

CUPS 打印机驱动程序必然取决于 CUPS 软件的特定文件格式和特定实现细节。请参阅 CUPS.org 了解更多信息。

本文件中使用的术语

  目标 是接受打印任务的打印机或打印队列。 打印任务 是目标使用创建任务时提供的选项处理的一个或多个文档的集合。 文档 是适合打印的文件（ JPEG 图像、 PDF 文件等）。 选项 控制打印的某些方面，例如使用的介质。 介质 是打印在其上的纸张或卷。 属性 是为 Internet 打印协议（ IPP ）请求编码的选项。

编译使用 CUPS API 的程序

  CUPS 库可以从任何 C 、 C++ 或 Object C 程序中使用。根据库进行编译的方法因操作系统和 CUPS 的安装而异。以下各节介绍如何在两种常见环境中编译简单程序（如下所示）。

  以下简单程序列出了可用的目标： 

#include <stdio.h>
#include <cups/cups.h>

int print_dest(void *user_data, unsigned flags, cups_dest_t *dest)
{
  if (dest->instance)
    printf("%s/%s\n", dest->name, dest->instance);
  else
    puts(dest->name);

  return (1);
}

int main(void)
{
  cupsEnumDests(CUPS_DEST_FLAGS_NONE, 1000, NULL, 0, 0, print_dest, NULL);

  return (0);
}

用 Xcode 编译

  在 Xcode 中，从 文件 菜单中选择 新建项目… （或按 SHIFT+CMD+N ），然后选择 macOS 应用程序项目类型下的 命令行工具 。单击 下一步 并输入项目名称，例如 “firstcups” 。单击 下一步 并选择项目目录。单击 下一步 创建项目。

  在项目窗口中，单击 构建阶段 组并展开 二进制库部分的链接 。单击 + ，键入 “libcups” 显示库，然后双击 libcups.tbd 。

  最后，单击侧栏中的 main.c 文件，并将示例程序复制到该文件中。构建并运行（ CMD+R ）以查看目标列表。

用 GCC 编译

  在命令行中，使用您喜爱的编辑器创建一个名为 simple.c 的文件，将示例复制到此文件，然后保存。然后运行以下命令，用 GCC 编译并运行它： 


gcc -o simple `cups-config --cflags` simple.c `cups-config --libs`
./simple

  cups-config 命令提供本地系统所需的编译器标志（ cups-config --cflags ）和库（ cups-config --libs ）。

使目标工作

  CUPS 中的目标表示单个打印机或打印机类（集合或池），由 cups_dest_t 结构表示，该结构包括名称（ name ）、实例（ instance 、保存的选项/设置），无论目标是否为用户的默认值（ is_default ），以及与该目标相关联的选项和基本信息（ num_options 和 options ）。

  历史上，目标由系统或网络管理员手动维护，但 CUPS 还支持在当前网络上动态发现目标。

查找可用目标

  cupsEnumDests 函数查找所有可用的目标： 


int cupsEnumDests(unsigned flags, int msec, int *cancel,
                  cups_ptype_t type, cups_ptype_t mask,
                  cups_dest_cb_t cb, void *user_data);

  flags 参数指定枚举选项，当前必须是 CUPS_DEST_FLAGS_NONE。

  msec 参数指定枚举应使用的最大时间量（毫秒） － 交互式应用程序在主线程上运行时应将此值保留在 5000 或更少。

  cancel 参数指向一个整型变量，当该变量设置为非零值时，将导致枚举尽快停止。如果不需要，它可以是 NULL 。

  type 和 mask 参数是允许调用者根据类别和/或函数筛选目标的位字段。目标的 “printer-type” 值被 mask 值屏蔽，并在过滤时与 type 值进行比较。例如，要仅枚举本地系统上承载的目标，请对 type 参数传递 CUPS_PRINTER_LOCAL ，对 mask 参数传递 CUPS_PRINTER_DISCOVERED 。以下常量可用于筛选：

  • CUPS_PRINTER_CLASS: 目标的集合。
  • CUPS_PRINTER_FAX: 传真机。
  • CUPS_PRINTER_LOCAL: 本地打印机或类。此常量的值为 0 （未设置位），仅用于 type 参数，并与 CUPS_PRINTER_REMOTE 或 CUPS_PRINTER_DISCOVERED 常量通过 mask 参数传递。
  • CUPS_PRINTER_REMOTE: 远程（共享）打印机或类。
  • CUPS_PRINTER_DISCOVERED: 可用的网络打印机或类。
  • CUPS_PRINTER_BW: 可以 B&W 印刷。
  • CUPS_PRINTER_COLOR: 可以彩色印刷。
  • CUPS_PRINTER_DUPLEX: 可以双面打印。
  • CUPS_PRINTER_STAPLE: 可以装订输出。
  • CUPS_PRINTER_COLLATE: 可以快速整理副本。
  • CUPS_PRINTER_PUNCH: 可以打孔输出。
  • CUPS_PRINTER_COVER: 可以覆盖输出。
  • CUPS_PRINTER_BIND: 可以绑定输出。
  • CUPS_PRINTER_SORT: 可以对输出（邮箱等）进行排序。
  • CUPS_PRINTER_SMALL: 可以打印 Letter/Legal/A4-size 介质。
  • CUPS_PRINTER_MEDIUM: 可以打印 Tabloid/B/C/A3/A2-size 介质。
  • CUPS_PRINTER_LARGE: 可以打印 D/E/A1/A0-size 介质。
  • CUPS_PRINTER_VARIABLE: 可以在卷筒和自定义大小的介质上打印。

  cb 参数指定为找到的每个目标调用的函数： 


typedef int (*cups_dest_cb_t)(void *user_data,
                              unsigned flags,
                              cups_dest_t *dest);

  回调函数接收 user_data 参数的副本以及位字段（ flags ）和找到的目标。 flags 参数可以设置以下任何常量（位）值：

  • CUPS_DEST_FLAGS_MORE: 还有更多的目标。
  • CUPS_DEST_FLAGS_REMOVED: 目标已离开，应从用户可以选择的目标列表中删除。
  • CUPS_DEST_FLAGS_ERROR: 发生了一个错误。通过调用 cupsLastError 和/或 cupsLastErrorString 函数可以找到错误原因。

  回调函数返回 0 以停止枚举，或返回 1 以继续。

提示：

对于同一个目标，回调函数可能会被多次调用，因此由调用方来抑制任何重复的目标。

  以下示例显示了如何使用 cupsEnumDests 获取目标的筛选数组： 


typedef struct
{
  int num_dests;
  cups_dest_t *dests;
} my_user_data_t;

int my_dest_cb(my_user_data_t *user_data, unsigned flags,
               cups_dest_t *dest)
{
  if (flags & CUPS_DEST_FLAGS_REMOVED)
  {
   /*
    * Remove destination from array...
    */

    user_data->num_dests =
        cupsRemoveDest(dest->name, dest->instance,
                       user_data->num_dests,
                       &(user_data->dests));
  }
  else
  {
   /*
    * Add destination to array...
    */

    user_data->num_dests =
        cupsCopyDest(dest, user_data->num_dests,
                     &(user_data->dests));
  }

  return (1);
}

int my_get_dests(cups_ptype_t type, cups_ptype_t mask,
                 cups_dest_t **dests)

  my_user_data_t user_data = { 0, NULL };

基本目标信息

  cups_dest_t 结构体的 num_options 和 options 成员除了提供该目标的用户默认选项和值外，还提供有关该目标的基本属性。以下名称是为各种目标属性预定义的：

  • "auth-info-required": 打印到此目标所需的身份验证类型：“none” , “username,password” ， “domain,username,password” 或 “negotiate”（ Kerberos ）。
  • "printer-info": 目标的可读描述，如“我的激光打印机”。
  • "printer-is-accepting-jobs": "true" if the destination is accepting new jobs, "false" otherwise.
  • "printer-is-shared": 如果目标与其他计算机共享，则为 “true” ，否则为 “false” 。
  • "printer-location": 目标的可读位置，如 “实验室 4” 。
  • "printer-make-and-model": 目标的可读品牌和型号，如 “ExampleCorp LaserPrinter 4000 系列” 。
  • "printer-state": 如果目标处于空闲状态，则为 “3” ；如果目标正在打印任务，则为 “4” ；如果目标已停止，则为 “5” 。
  • "printer-state-change-time": 目标进入当前状态时的 UNIX 时间。
  • "printer-state-reasons": 目标的其他逗号分隔状态关键字，如 “media-tray-empty-error” 和 “toner-low-warning” 。
  • "printer-type": cups_ptype_t 与目标关联的值。
  • "printer-uri-supported": 与目标相关联的 URI ；如果未设置，则已找到此目标，但尚未设置为本地打印机。

  使用 cupsGetOption 函数检索值。例如，以下代码获取目标的品牌和型号： 

const char *model = cupsGetOption("printer-make-and-model",
                                  dest->num_options,
                                  dest->options);

详细目标信息

  选择目标后，可以使用 cupsCopyDestInfo 函数收集有关目标的详细信息： 

cups_dinfo_t *
cupsCopyDestInfo(http_t *http, cups_dest_t *dest);

  http 参数指定与 CUPS 调度程序的连接，通常为常量 CUPS_HTTP_DEFAULT 。 dest 参数指定要查询的目标。

  cups_dinfo_t 返回的结构体包含受支持选项及其支持、就绪和默认值的快照。它还可以报告不同选项和值之间的约束，并建议更改以解决这些约束。

获取支持的选项和值

  cupsCheckDestSupported 函数可用于测试是否支持特定选项或选项和值： 

int
cupsCheckDestSupported(http_t *http, cups_dest_t *dest,
                       cups_dinfo_t *info,
                       const char *option,
                       const char *value);

  option 参数指定要检查的选项的名称。以下常量可用于检查各种标准选项：

  • CUPS_COPIES: 控制生成的副本数。
  • CUPS_FINISHINGS: 以逗号分隔的整数常量列表，用于控制应用于任务的精加工过程，包括装订、打孔和折叠。
  • CUPS_MEDIA: 控制所使用的介质大小，通常为以下之一： CUPS_MEDIA_3X5 ， CUPS_MEDIA_4X6 ， CUPS_MEDIA_5X7 ， CUPS_MEDIA_8X10 ， CUPS_MEDIA_A3 ， CUPS_MEDIA_A4 ， CUPS_MEDIA_A5 ， CUPS_MEDIA_A6 ， CUPS_MEDIA_ENV10 ， CUPS_MEDIA_ENVDL ， CUPS_MEDIA_LEGAL ， CUPS_MEDIA_LETTER ， CUPS_MEDIA_PHOTO_L ， CUPS_MEDIA_SUPERBA3 或 CUPS_MEDIA_TABLOID 。
  • CUPS_MEDIA_SOURCE: 控制从何处取出介质，通常为 CUPS_MEDIA_SOURCE_AUTO 或 CUPS_MEDIA_SOURCE_MANUAL 。
  • CUPS_MEDIA_TYPE: 控制所使用的介质类型，通常为以下之一： CUPS_MEDIA_TYPE_AUTO ， CUPS_MEDIA_TYPE_ENVELOPE ， CUPS_MEDIA_TYPE_LABELS ， CUPS_MEDIA_TYPE_LETTERHEAD ， CUPS_MEDIA_TYPE_PHOTO ， CUPS_MEDIA_TYPE_PHOTO_GLOSSY ， CUPS_MEDIA_TYPE_PHOTO_MATTE ， CUPS_MEDIA_TYPE_PLAIN 或 CUPS_MEDIA_TYPE_TRANSPARENCY 。
  • CUPS_NUMBER_UP: 控制放置在每个介质侧的文档页数。
  • CUPS_ORIENTATION: 控制放置在介质上的文档页面的方向： CUPS_ORIENTATION_PORTRAIT 或 CUPS_ORIENTATION_LANDSCAPE。
  • CUPS_PRINT_COLOR_MODE: 控制输出是彩色（ CUPS_PRINT_COLOR_MODE_COLOR ）、灰度（ CUPS_PRINT_COLOR_MODE_MONOCHROME ）还是（ CUPS_PRINT_COLOR_MODE_AUTO ）。
  • CUPS_PRINT_QUALITY: 控制输出的生成质量： CUPS_PRINT_QUALITY_DRAFT ， CUPS_PRINT_QUALITY_NORMAL 或 CUPS_PRINT_QUALITY_HIGH 。
  • CUPS_SIDES: 控制打印是放置在介质的一侧还是两侧： CUPS_SIDES_ONE_SIDED ， CUPS_SIDES_TWO_SIDED_PORTRAIT 或 CUPS_SIDES_TWO_SIDED_LANDSCAPE 。

  如果 value 参数为 NULL ，则 cupsCheckDestSupported 函数返回目标是否支持该选项。否则，函数将返回是否支持该选项的指定值。

  cupsFindDestSupported 函数返回包含给定选项支持值的 IPP 属性： 

 ipp_attribute_t *
 cupsFindDestSupported(http_t *http, cups_dest_t *dest,
                       cups_dinfo_t *dinfo,
                       const char *option);

  例如，以下代码将目标（如果有）支持的完成过程打印到标准输出： 

cups_dinfo_t *info = cupsCopyDestInfo(CUPS_HTTP_DEFAULT,
                                      dest);

if (cupsCheckDestSupported(CUPS_HTTP_DEFAULT, dest, info,
                           CUPS_FINISHINGS, NULL))
{
  ipp_attribute_t *finishings =
      cupsFindDestSupported(CUPS_HTTP_DEFAULT, dest, info,
                            CUPS_FINISHINGS);
  int i, count = ippGetCount(finishings);

  puts("finishings supported:");
  for (i = 0; i < count; i ++)
    printf("  %d\n", ippGetInteger(finishings, i));
}
else
  puts("finishings not supported.");

  可以查询 “job-creation-attributes” 选项，以获得受支持选项的列表。例如，以下代码将支持的选项列表打印到标准输出： 

ipp_attribute_t *attrs =
    cupsFindDestSupported(CUPS_HTTP_DEFAULT, dest, info,
                          "job-creation-attributes");
int i, count = ippGetCount(attrs);

for (i = 0; i < count; i ++)
  puts(ippGetString(attrs, i, NULL));

获取默认值

  有两组默认值 － 用户默认值可通过 num_options 和 options 成员的 cups_dest_t 结构体，并且目标默认值可通过 cups_dinfo_t 结构体和 cupsFindDestDefault 函数获得，该函数返回 IPP 属性，包含给定选项的默认值： 

ipp_attribute_t *
cupsFindDestDefault(http_t *http, cups_dest_t *dest,
                    cups_dinfo_t *dinfo,
                    const char *option);

  cupsGetOption 中的用户默认值应始终优先于目标默认值。例如，以下代码将默认 finishings 值打印到标准输出： 

const char *def_value =
    cupsGetOption(CUPS_FINISHINGS, dest->num_options,
                  dest->options);
ipp_attribute_t *def_attr =
    cupsFindDestDefault(CUPS_HTTP_DEFAULT, dest, info,
                        CUPS_FINISHINGS);

if (def_value != NULL)
{
  printf("Default finishings: %s\n", def_value);
}
else
{
  int i, count = ippGetCount(def_attr);

  printf("Default finishings: %d",
         ippGetInteger(def_attr, 0));
  for (i = 1; i < count; i ++)
    printf(",%d", ippGetInteger(def_attr, i));
  putchar('\n');
}

获取已就绪（加载）值

  finishings 和 media 选项还支持查询就绪或加载的值。例如，打印机可能安装了打孔和装订修整器，但没有装订 － 支持的值将列出打孔和装订修整过程，但就绪值将仅列出打孔过程。类似地，打印机可能支持数百种不同大小的介质，但在任何给定时间都只能加载一种大小的介质 － 就绪值仅限于打印机中实际存在的介质。

  cupsfinddesready 函数查找包含给定选项的就绪值的 IPP 属性： 

ipp_attribute_t *
cupsFindDestReady(http_t *http, cups_dest_t *dest,
                  cups_dinfo_t *dinfo, const char *option);

  例如，以下代码列出了准备完成的过程： 

ipp_attribute_t *ready_finishings =
    cupsFindDestReady(CUPS_HTTP_DEFAULT, dest, info,
                      CUPS_FINISHINGS);

if (ready_finishings != NULL)
{
  int i, count = ippGetCount(ready_finishings);

  puts("finishings ready:");
  for (i = 0; i < count; i ++)
    printf("  %d\n", ippGetInteger(ready_finishings, i));
}
else
  puts("no finishings are ready.");

介质尺寸选项

  CUPS 提供了查询每个支持的介质尺寸选项的尺寸和边距的函数。 cups_size_t 结构体用于描述介质尺寸： 

typedef struct cups_size_s
{
  char media[128];
  int width, length;
  int bottom, left, right, top;
} cups_size_t;

  width 和 length 成员以百分之一毫米（ 1/2540 英寸）为单位指定介质的尺寸。bottom 、 left 、 right 和 top 成员指定可打印区域的边距，也以百分之一毫米为单位。

  cupsGetDestMediaByName 和 cupsGetDestMediaBySize 函数使用标准介质尺寸名称或百分之一毫米的尺寸查找介质尺寸信息： 

int
cupsGetDestMediaByName(http_t *http, cups_dest_t *dest,
                       cups_dinfo_t *dinfo,
                       const char *media,
                       unsigned flags, cups_size_t *size);

int
cupsGetDestMediaBySize(http_t *http, cups_dest_t *dest,
                       cups_dinfo_t *dinfo,
                       int width, int length,
                       unsigned flags, cups_size_t *size);

  media 、 width 和 length 参数指定要查找的大小。 flags 参数指定控制各种查找选项的位字段：

  • CUPS_MEDIA_FLAGS_DEFAULT: 查找打印机支持的最接近的尺寸。
  • CUPS_MEDIA_FLAGS_BORDERLESS: 找到一个无边界的尺寸。
  • CUPS_MEDIA_FLAGS_DUPLEX: 找到与双面打印兼容的尺寸。
  • CUPS_MEDIA_FLAGS_EXACT: 找到与尺寸完全匹配的。
  • CUPS_MEDIA_FLAGS_READY: 如果打印机支持介质感应或配置每个纸盘/纸源中的介质，请在 “ready” 介质中查找尺寸。

  如果找到目标的匹配大小，则大小信息存储在 size 参数所指向的结构体中，并返回 1 。否则返回 0 。

  例如，以下代码在 US Letter 介质上打印双面打印的页边距： 

cups_size_t size;

if (cupsGetDestMediaByName(CUPS_HTTP_DEFAULT, dest, info,
                           CUPS_MEDIA_LETTER,
                           CUPS_MEDIA_FLAGS_DUPLEX, &size))
{
  puts("Margins for duplex US Letter:");
  printf("  Bottom: %.2fin\n", size.bottom / 2540.0);
  printf("    Left: %.2fin\n", size.left / 2540.0);
  printf("   Right: %.2fin\n", size.right / 2540.0);
  printf("     Top: %.2fin\n", size.top / 2540.0);
}
else
  puts("Margins for duplex US Letter are not available.");

  您还可以使用 cupsGetDestMediaByIndex 和 cupsGetDestMediaCount 函数枚举与给定 flags 值匹配的所有尺寸： 

int
cupsGetDestMediaByIndex(http_t *http, cups_dest_t *dest,
                        cups_dinfo_t *dinfo, int n,
                        unsigned flags, cups_size_t *size);

int
cupsGetDestMediaCount(http_t *http, cups_dest_t *dest,
                      cups_dinfo_t *dinfo, unsigned flags);

  例如，以下代码打印就绪介质列表和相应的页边距： 

cups_size_t size;
int i;
int count = cupsGetDestMediaCount(CUPS_HTTP_DEFAULT,
                                  dest, info,
                                  CUPS_MEDIA_FLAGS_READY);

for (i = 0; i < count; i ++)
{
  if (cupsGetDestMediaByIndex(CUPS_HTTP_DEFAULT, dest, info,
                              i, CUPS_MEDIA_FLAGS_READY,
                              &size))
  {
    printf("%s:\n", size.name);
    printf("   Width: %.2fin\n", size.width / 2540.0);
    printf("  Length: %.2fin\n", size.length / 2540.0);
    printf("  Bottom: %.2fin\n", size.bottom / 2540.0);
    printf("    Left: %.2fin\n", size.left / 2540.0);
    printf("   Right: %.2fin\n", size.right / 2540.0);
    printf("     Top: %.2fin\n", size.top / 2540.0);
  }
}

  最后， cupsGetDestMediaDefault 函数返回默认的介质尺寸： 

int
cupsGetDestMediaDefault(http_t *http, cups_dest_t *dest,
                        cups_dinfo_t *dinfo, unsigned flags,
                        cups_size_t *size);

提交打印任务

  准备好提交打印任务后，可使用 cupsCreateDestJob 函数创建任务： 

ipp_status_t
cupsCreateDestJob(http_t *http, cups_dest_t *dest,
                  cups_dinfo_t *info, int *job_id,
                  const char *title, int num_options,
                  cups_option_t *options);

  title 参数指定打印任务的名称，例如“我的文档”。num_options 和 options 参数指定使用 cupsAddOption 函数分配的打印任务选项。

  成功时，任务的数字标识符存储在 job_id 参数指向的整数中，并返回 IPP_STATUS_OK 。否则，将返回 IPP 错误状态。

  例如，以下代码创建一个新任务，该任务将打印 42 份双面 US Letter 文档： 

int job_id = 0;
int num_options = 0;
cups_option_t *options = NULL;

num_options = cupsAddOption(CUPS_COPIES, "42",
                            num_options, &options);
num_options = cupsAddOption(CUPS_MEDIA, CUPS_MEDIA_LETTER,
                            num_options, &options);
num_options = cupsAddOption(CUPS_SIDES,
                            CUPS_SIDES_TWO_SIDED_PORTRAIT,
                            num_options, &options);

if (cupsCreateDestJob(CUPS_HTTP_DEFAULT, dest, info,
                      &job_id, "My Document", num_options,
                      options) == IPP_STATUS_OK)
  printf("Created job: %d\n", job_id);
else
  printf("Unable to create job: %s\n",
         cupsLastErrorString());

  创建任务后，您可以使用 cupsStartDestDocument 、 cupsWriteRequestData 和 cupsFinishDestDocument 函数提交任务的文档： 

http_status_t
cupsStartDestDocument(http_t *http, cups_dest_t *dest,
                      cups_dinfo_t *info, int job_id,
                      const char *docname,
                      const char *format,
                      int num_options,
                      cups_option_t *options,
                      int last_document);

http_status_t
cupsWriteRequestData(http_t *http, const char *buffer,
                     size_t length);

ipp_status_t
cupsFinishDestDocument(http_t *http, cups_dest_t *dest,
                       cups_dinfo_t *info);

  docname 参数指定文档的名称，通常是原始文件名。 format 参数指定文档的 MIME 媒体类型，包括以下常量：

  • CUPS_FORMAT_JPEG: "image/jpeg"
  • CUPS_FORMAT_PDF: "application/pdf"
  • CUPS_FORMAT_POSTSCRIPT: "application/postscript"
  • CUPS_FORMAT_TEXT: "text/plain"

  num_options 和 options 参数指定每个文档的打印选项，当前必须为 0 和 NULL 。 last_document 参数指定这是否是任务中的最后一个文档。

  例如，以下代码向刚创建的任务提交 PDF 文件： 

FILE *fp = fopen("filename.pdf", "rb");
size_t bytes;
char buffer[65536];

if (cupsStartDestDocument(CUPS_HTTP_DEFAULT, dest, info,
                          job_id, "filename.pdf", 0, NULL,
                          1) == HTTP_STATUS_CONTINUE)
{
  while ((bytes = fread(buffer, 1, sizeof(buffer), fp)) > 0)
    if (cupsWriteRequestData(CUPS_HTTP_DEFAULT, buffer,
                             bytes) != HTTP_STATUS_CONTINUE)
      break;

  if (cupsFinishDestDocument(CUPS_HTTP_DEFAULT, dest,
                             info) == IPP_STATUS_OK)
    puts("Document send succeeded.");
  else
    printf("Document send failed: %s\n",
           cupsLastErrorString());
}

fclose(fp);

发送 IPP 请求

  CUPS 提供了一个丰富的 API ，用于向调度程序或打印机发送 IPP 请求，通常来自主要目的不是发送打印任务的管理或实用程序应用程序。

连接到调度程序或打印机

  与调度程序或打印机的连接由 HTTP 连接类型 http_t 表示。 cupsConnectDest 函数连接到与目标关联的调度程序或打印机： 

http_t *
cupsConnectDest(cups_dest_t *dest, unsigned flags, int msec,
                int *cancel, char *resource,
                size_t resourcesize, cups_dest_cb_t cb,
                void *user_data);

  dest 参数指定要连接的目标。

  flags 参数指定是否要连接到与目标关联的调度程序（ CUPS_DEST_FLAGS_NONE ）或设备/打印机（ CUPS_DEST_FLAGS_DEVICE ）。

  msec 参数指定您愿意等待连接建立的时间（以毫秒为单位）。指定值 -1 以无限期等待。

  cancel 参数指定可设置为非零值以取消连接的整数变量的地址。指定 NULL 值以不提供取消变量。

  resource 和 resourcesize 参数指定字符串数组的地址和大小，以保存发送 IPP 请求时使用的路径。

  cb 和 user_data 参数指定一个目标回调函数，该函数返回 1 以继续连接，返回 0 以停止连接。目标回调的工作方式与用于 cupsEnumDests 函数的回调相同。

  成功后，将返回一个 HTTP 连接，该连接可用于发送 IPP 请求和获取 IPP 响应。

  例如，以下代码以 30 秒超时连接到与目标关联的打印机： 

char resource[256];
http_t *http = cupsConnectDest(dest, CUPS_DEST_FLAGS_DEVICE,
                               30000, NULL, resource,
                               sizeof(resource), NULL, NULL);

创建 IPP 请求

  IPP 请求由 IPP 消息类型 ipp_t 表示，请求中的每个 IPP 属性都使用类型 ipp_attribute_t 表示。每个 IPP 请求包括一个操作代码（ IPP_OP_CREATE_JOB ， IPP_OP_GET_PRINTER_ATTRIBUTES 等）和一个 32 位整数标识符。

  ippNewRequest 函数创建一个新的 IPP 请求： 

ipp_t *
ippNewRequest(ipp_op_t op);

  op 参数指定请求的 IPP 操作代码。例如，以下代码创建 IPP 获取打印机属性请求： 

ipp_t *request = ippNewRequest(IPP_OP_GET_PRINTER_ATTRIBUTES);

  请求标识符将自动设置为当前进程的唯一值。

  每个 IPP 请求都以两个 IPP 属性开始，“attributes-charset” 和 “attributes-natural-language” ，然后是指定操作目标的 IPP 属性。ippNewRequest 自动添加正确的 “attributes-charset” 和 “attributes-natural-language” 属性，但您必须添加目标属性。例如，以下代码将 “printer-uri” 属性添加到 IPP Get-Printer-Attributes 请求中，以指定要查询的打印机： 

const char *printer_uri = cupsGetOption("device-uri",
                                        dest->num_options,
                                        dest->options);

ippAddString(request, IPP_TAG_OPERATION, IPP_TAG_URI,
             "printer-uri", NULL, printer_uri);

提示：

如果我们想查询调度程序而不是设备，我们将查找 “printer-uri-supported” 选项而不是 “device-uri” 值。

  ippAddString 函数将 “printer-uri” 属性添加到 IPP 请求中。IPP_TAG_OPERATION 参数指定属性是操作的一部分。 IPP_TAG_URI 参数指定该值为通用资源标识符（ URI ）字符串。 NULL 参数指定没有与字符串关联的语言（英语、法语、日语等），而 printer_uri 参数指定字符串值。

  IPP 获取打印机属性请求还支持名为 “requested-attributes” 的 IPP 属性，该属性列出了您感兴趣的属性和值。例如，以下代码请求打印机状态属性： 

static const char * const requested_attributes[] =
{
  "printer-state",
  "printer-state-message",
  "printer-state-reasons"
};

ippAddStrings(request, IPP_TAG_OPERATION, IPP_TAG_KEYWORD,
              "requested-attributes", 3, NULL,
              requested_attributes);

  ippAddStrings 函数使用一个或多个字符串添加属性，在本例中为三个字符串。IPP_TAG_KEYWORD 参数指定字符串是用于属性名称的关键字值。所有字符串都使用相同的语言（ NULL ），该属性将包含数组 requested_attributes 中的三个字符串。

  CUPS 提供了许多函数来添加不同类型的属性：

  • ippAddBoolean 使用一个值添加布尔（ IPP_TAG_BOOLEAN ）属性。
  • ippAddInteger 使用一个值添加 enum （ IPP_TAG_INTEGER ）或 integer （ IPP_TAG_INTEGER ）属性。
  • ippAddIntegers 添加具有一个或多个值的枚举或整数属性。
  • ippAddOctetString 添加带有一个值的 octetString 属性。
  • ippAddOutOfBand 添加 admin-defined （ IPP_TAG_ADMINDEFINE ）、 default （ IPP_TAG_DEFAULT ）、 delete-attribute （ IPP_TAG_DELETEATTR ）、 no-value （ IPP_TAG_NOVALUE ）、 not-settable （ IPP_TAG_NOTSETTABLE ）、 unknown （ IPP_TAG_UNKNOWN ）或 unsupported （ IPP_TAG_UNSUPPORTED_VALUE ）带外属性。
  • ippAddRange 添加带有一个范围的 rangeOfInteger 属性。
  • ippAddRanges 添加带有一个或多个范围的 rangeOfInteger 属性。
  • ippAddResolution 添加具有一个分辨率的 resolution 属性。
  • ippAddResolutions 添加具有一个或多个分辨率的 resolution 属性。
  • ippAddString 添加 charset （ IPP_TAG_CHARSET ）、 keyword （ IPP_TAG_KEYWORD ）、 mimeMediaType （ IPP_TAG_MIMETYPE ）、 name （ IPP_TAG_NAME 和 IPP_TAG_NAMELANG ）、 naturalLanguage （ IPP_TAG_NATURAL_LANGUAGE ）、 text （ IPP_TAG_TEXT 和 IPP_TAG_TEXTLANG ）、 uri （ IPP_TAG_URI ）或 uriScheme （ IPP_TAG_URISCHEME ）属性的一个值。
  • ippAddStrings 使用一个或多个值添加 charset 、 keyword 、 mimeMediaType 、 name 、 naturalLanguage 、 text 、 uri 或 uriScheme 属性。

发送 IPP 请求

  创建 IPP 请求后，可以使用 cupsDoRequest 函数发送请求。例如，以下代码将 IPP Get-Printer-Attributes 请求发送到目标并保存响应： 

ipp_t *response = cupsDoRequest(http, request, resource);

  对于包含文件的发送文档等请求，应使用 cupsDoFileRequest 函数： 

ipp_t *response = cupsDoFileRequest(http, request, resource,
                                    filename);

  cupsDoRequest 和 cupsDoFileRequest 都可以释放 IPP 请求。如果收到有效的 IPP 响应，它将存储在新的 IPP 消息（ ipp_t ）中，并返回给调用者。否则将返回 NULL 。

  可以使用 cupsLastError 函数查询最近请求的状态，例如： 

if (cupsLastError() >= IPP_STATUS_ERROR_BAD_REQUEST)
{
  /* request failed */
}

  还可以使用 cupsLastErrorString 函数获取人类可读的错误消息： 

if (cupsLastError() >= IPP_STATUS_ERROR_BAD_REQUEST)
{
  /* request failed */
  printf("Request failed: %s\n", cupsLastErrorString());
}

处理 IPP 响应

  对 IPP 请求的每个响应也是一条 IPP 消息（ ipp_t ），它有自己的 IPP 属性（ ipp_attribute_t ），其中包括一个状态代码（ IPP_STATUS_OK ， IPP_STATUS_ERROR_BAD_REQUEST ），以及请求中相应的 32 位整数标识符。

  例如，以下代码查找打印机状态属性并打印其值： 

ipp_attribute_t *attr;

if ((attr = ippFindAttribute(response, "printer-state",
                             IPP_TAG_ENUM)) != NULL)
{
  printf("printer-state=%s\n",
         ippEnumString("printer-state", ippGetInteger(attr, 0)));
}
else
  puts("printer-state=unknown");

if ((attr = ippFindAttribute(response, "printer-state-message",
                             IPP_TAG_TEXT)) != NULL)
{
  printf("printer-state-message=\"%s\"\n",
         ippGetString(attr, 0, NULL)));
}

if ((attr = ippFindAttribute(response, "printer-state-reasons",
                             IPP_TAG_KEYWORD)) != NULL)
{
  int i, count = ippGetCount(attr);

  puts("printer-state-reasons=");
  for (i = 0; i < count; i ++)
    printf("    %s\n", ippGetString(attr, i, NULL)));
}

  ippGetCount 函数返回属性中的数字值。

  ippGetInteger 和 ippGetString 函数从属性返回单个整数或字符串值。

  ippEnumString 函数将枚举值转换为其等价的关键字（字符串）。

  使用 IPP 响应消息后，使用 ippDelete 函数释放该消息： 

ippDelete(response);
