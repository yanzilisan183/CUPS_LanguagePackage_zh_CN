cupsenable(8)

名称

  cupsdisable, cupsenable - 停止/启动打印机和类

摘要

  cupsdisable [-E] [-U username] [ -c] [-h server[:port]] [-r reason] [ --hold] destination(s)
  cupsenable [-E] [-U username] [ -c] [-h server[:port]] [--release] destination(s)

说明

  cupsenable 启动命名打印机或类，而 cupsdisable 停止命名打印机或类。

选项

  可使用以下选项：

-E
强制加密与服务器的连接。
-U username
连接到服务器时使用指定的用户名。
-c
取消指定目标上的所有任务。
-h server[:port]
使用指定的服务器和端口。
--hold
保留指定打印机上的剩余任务。用于在执行维护之前完成当前任务。
-r "reason"
设置与停止状态关联的消息。如果未指定原因，则消息将设置为 “Reason Unknown” 。
--release
释放待打印的任务。在运行 cupsdisable 后使用 --hold 选项恢复打印。

遵循

  与 System V 打印系统不同， CUPS 允许打印机名称包含除空格、制表符、 “/” 或 “#” 之外的任何可打印字符。此外，打印机名和类名不区分大小写。

  这些命令的 System V 版本分别为 disable 和 enable 。它们已重命名，以避免与同名的 bash(1) 内置命令冲突。

  根据打印系统配置， CUPS 版本的 disable 和 enable 可能会要求用户输入访问密码。这与 System V 版本不同，后者要求 root 用户执行这些命令。
