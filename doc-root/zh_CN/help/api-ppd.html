概述

提示：

  PPD API 在 CUPS 1.6/macOS 10.8 中被弃用。请使用 CUPS 编程手册文档中的新工作通知单 API 。这些功能将在 CUPS 的未来版本中删除。

  CUPS PPD API 提供对 PostScript 打印机说明（ “PPD” ）文件中数据的只读访问，该文件用于所有带驱动程序的打印机。使用它，您可以获得向用户显示打印机选项所需的数据，标记选项选项并检查有无冲突的选项，以及在 PostScript 输出中输出标记的选项。 ppd_file_t 结构包含 PPD 文件中的所有信息。

提示：

  CUPS PPD API 使用术语 “option” 和 “choice” 而不是 Adobe 术语 “MainKeyword” 和 “OptionKeyword” 来表示特定的打印机选项和功能。 CUPS 还将 option（ “MainKeyword” ）和 choice（ “OptionKeyword” ）值视为不区分大小写的字符串，因此 option “InputSlot” 和 choice “Upper” 分别等效于 “inputslot” 和 “upper” 。

加载 PPD 文件

  ppdOpenFile 函数“打开”一个 PPD 文件并将其加载到内存中。例如，以下代码在 CUPS 筛选器中打开当前打印机的 PPD 文件： 

#include <cups/ppd.h>

ppd_file_t *ppd = ppdOpenFile(getenv("PPD"));

  返回值是指向新 ppd_file_t 结构的指针，如果 PPD 文件不存在或无法加载，则返回 NULL 。 ppdClose 函数释放结构体使用的内存： 

#include <cups/ppd.h>

ppd_file_t *ppd;

ppdClose(ppd);

  关闭后，指向 ppd_file_t 结构的指针将不再有效。

选项和组

  PPD 文件支持多个选项，这些选项存储在 ppd_option_t 和 ppd_choice_t 结构的数组中。

  每个选项依次与存储在 ppd_group_t 结构中的组相关联。可以在 PPD 文件中指定组；如果选项未与组关联，则将其放入自动生成的 “General” 组中。组也可以有子组，但是 CUPS 目前会忽略子组，因为以前滥用了此功能。

  通过使用三个功能之一对选项进行标记来选择选项。第一个是 ppdMarkDefaults ，它选择 PPD 文件中的所有默认选项： 

#include <cups/ppd.h>

ppd_file_t *ppd;

ppdMarkDefaults(ppd);

  第二个是 ppdMarkOption ，它在 PPD 文件中选择单个选项。例如，以下代码选择上部纸盒： 

#include <cups/ppd.h>

ppd_file_t *ppd;

ppdMarkOption(ppd, "InputSlot", "Upper");

  最后一个函数是 cupsMarkOptions ，它从 CUPS 选项数组中选择 PPD 文件中的多个选项，将 IPP 属性（如 “media” 和 “sides” ）映射到它们对应的 PPD 选项。您通常在打印筛选器中使用此功能与 cupsParseOptions 和 ppdMarkDefaults 一起选择任务所需的所有选项，例如： 

#include <cups/ppd.h>

ppd_file_t *ppd = ppdOpenFile(getenv("PPD"));
cups_option_t *options = NULL;
int num_options = cupsParseOptions(argv[5], 0, &options);

ppdMarkDefaults(ppd);
cupsMarkOptions(ppd, num_options, options);
cupsFreeOptions(num_options, options);

约束条件

  PPD 文件支持指定不同选项之间的冲突条件（称为约束）。约束存储在 ppd_const_t 结构的数组中，这些结构指定了相互冲突的选项和选择。 ppdConflicts 函数会告诉您所选选项中有多少不兼容。由于约束通常成对指定，因此返回值通常为偶数。

页面大小

  页面大小是一种特殊选项，具有与之相关联的物理尺寸和页边距。大小信息存储在 ppd_size_t 结构中，通过使用 ppdPageSize 函数查找命名大小即可获得。页面大小和边距以称为点的单位返回；每英寸有 72 个点。如果对大小传递 NULL ，则返回当前选择的大小： 

#include <cups/ppd.h>

ppd_file_t *ppd;
ppd_size_t *size = ppdPageSize(ppd, NULL);

  除了 PPD 文件中列出的标准页面大小外，有些打印机还支持可变或自定义页面大小。如果 ppd_file_t 结构的 variables_size 成员为非零，则支持自定义页面大小。 ppd_file_t 结构的 custom_min 、 custom_max 和 custom_margins 成员定义了可打印区域的限制。要得到介质尺寸，请使用 “Custom.widthxlength” 格式的页面大小字符串，其中 “width” 和 “length” 以点为单位。自定义页面大小名称也可以以英寸（ “Custom.widthxheightin” ）、厘米（ “Custom.widthxheightcm” ）或毫米（ “Custom.widthxheightmm” ）为单位指定： 

#include <cups/ppd.h>

ppd_file_t *ppd;

/* 获取 576x720 点自定义页面大小 */
ppd_size_t *size = ppdPageSize(ppd, "Custom.576x720");

/* 获取 8x10 英寸自定义页面大小 */
ppd_size_t *size = ppdPageSize(ppd, "Custom.8x10in");

/* 获取 100x200 毫米的自定义页面大小 */
ppd_size_t *size = ppdPageSize(ppd, "Custom.100x200mm");

/* 获取 12.7x34.5 厘米的自定义页面大小 */
ppd_size_t *size = ppdPageSize(ppd, "Custom.12.7x34.5cm");

  如果 PPD 不支持可变页面大小，则 ppdPageSize 函数将返回 NULL 。

属性

  每个 PPD 文件都由一个或多个属性组成。这些属性中的大多数用于定义组、选项（ options ）、选项（ choices ）和页面大小，但是可能存在一些信息属性，您可以在程序或筛选器中访问这些属性。属性通常类似于 PPD 文件中的以下示例之一： 

*name: "value"
*name spec: "value"
*name spec/text: "value"

  ppdFindAttr 和 ppdFindNextAttr 函数分别使用给定的 “spec” 字符串查找命名属性的第一个和下一个实例，并返回 ppd_attr_t 结构。如果提供 NULL 说明符字符串，则将返回具有给定名称的所有属性。例如，以下代码列出了 PPD 文件中的所有 Product 属性： 

#include <cups/ppd.h>

ppd_file_t *ppd;
ppd_attr_t *attr;

for (attr = ppdFindAttr(ppd, "Product", NULL);
     attr != NULL;
     attr = ppdFindNextAttr(ppd, "Product", NULL))
  puts(attr->value);

参数

  获取指定服务器上打印机的 PPD 文件（如果已更改）。

http_status_t cupsGetPPD3(http_t *http, const char *name, time_t *modtime, char *buffer, size_t bufsize);

参数

http HTTP 连接或 CUPS_HTTP_DEFAULT
name 目标名称
modtime 修改时间
buffer 文件名缓冲区

 CUPS 1.2/macOS 10.5 cupsRasterInterpretPPD

  解释 PPD 命令以创建页眉。

int cupsRasterInterpretPPD(cups_page_header2_t *h, ppd_file_t *ppd, int num_options, cups_option_t *options, cups_interpret_cb_t func);

参数

h 要创建的页眉
ppd PPD 文件
num_options 选择的数量
options 选项
func 可选的页眉回调（ NULL 表示无）

返回值

  成功为 0 ，失败为 -1

 CUPS 1.4/macOS 10.6 cupsResolveConflicts

  在标记的 PPD 中解决冲突。

int cupsResolveConflicts(ppd_file_t *ppd, const char *option, const char *choice, int *num_options, cups_option_t **options);

参数

ppd PPD 文件
option 新选择的 option 选项或NULL表示无
choice 新选择的 choice 选项或NULL表示无
num_options 其他选定选项的数目
options 其他选定选项

返回值

  成功为 1 ，失败为 0

详述

  此函数尝试解决标记 PPD 中的任何冲突，返回解决冲突所需的选项更改列表。输入时， “num_options” 和 “options” 包含尚未标记的任何未决选项更改，而 “option” 和 “choice” 包含最近的选择，这些选择可能在 “num_options” 或 “options” 中，也可能不在其中。

  成功返回后， “num_options” 和 “options” 将更新为包含 “option” 和 “choice” 以及解决 PPD 文件中指定的冲突所需的任何更改，并返回 1 。

  如果无法解决选项冲突，则不会更改 “num_options” 和 “options” ，并返回 0 。

  在解决冲突时，cupsResolveConflicts 不考虑对当前页大小的更改（ media ， PageSize 和 PageRegion ）或 “option” 中指定的最新选项。因此，如果解决冲突的唯一方法是更改页面大小或用户最近更改的选项， cupsResolveConflicts 将返回 0 ，表示无法解决冲突。

  cupsResolveConflicts 函数使用选项约束信息的两个来源之一。首选约束信息由 cupsUIConstraints 和 cupsUIResolver 属性定义 － 在这种情况下， PPD 文件提供约束解析操作。

  备份约束信息由 UIConstraints 和 NonUIConstraints 属性定义。这些约束通过算法解决，首先选择冲突选项的默认选项，然后迭代所有可能的选项，直到找到非冲突选项。
